# Инструкция по разметке новых снимков

## Когда нужна разметка

- Ухудшилось качество распознавания на новых условиях съёмки (другое освещение, камера, расстояние).
- Добавляются новые типы объектов.
- Перед fine-tuning на реальных фото (после синтетического обучения).

---

## Инструмент: X-AnyLabeling

Поддерживает OBB (Oriented Bounding Boxes) — повёрнутые прямоугольники для карт под углом.

```bash
# Создать отдельный venv в корне репозитория (из direnv-окружения с Python 3.11)
python3.11 -m venv .venv-anylabeling

# Установить x-anylabeling и незадекларированные зависимости (баги пакета)
.venv-anylabeling/bin/pip install x-anylabeling requests importlib_metadata

# Пофиксить баг x-anylabeling: label_colormap() возвращает read-only массив
sed -i 's/LABEL_COLORMAP = imgviz.label_colormap()/LABEL_COLORMAP = imgviz.label_colormap().copy()/' \
  .venv-anylabeling/lib/python3.11/site-packages/anylabeling/views/labeling/label_widget.py

# Запуск (direnv должен быть активен — он выставляет PYTHONPATH для _tkinter)
.venv-anylabeling/bin/anylabeling
```

> **NixOS**: tkinter требует `PYTHONPATH`, который выставляется через direnv (`direnv allow` в корне репо). Без активного direnv будет `No module named '_tkinter'`.

**Альтернатива без установки:** [Roboflow](https://roboflow.com) — web, поддерживает OBB, бесплатный tier.

## Пошаговая разметка в X-AnyLabeling

### 1. Открыть изображение

`File → Open Image` или `File → Open Dir` (для разметки папки).

### 2. Разметка нижних полосок карт (класс `card_strip`)

1. Нажать `R` (Rotated Bounding Box) или выбрать в меню.
2. Нарисовать **плотно прилегающий повёрнутый прямоугольник** вокруг нижней полоски карты.
   - Полоска — это видимая нижняя часть карты (скоринговая строка).
   - Захватывать только видимую полоску, **не** всю карту.
3. В поле Label ввести: `card_strip`
4. Повторить для каждой видимой полоски.

> **Важно:** если полоска почти полностью перекрыта другой картой (видно менее 20% ширины), **пропустить** её.

### 3. Разметка жетонов

Для каждого жетона нарисовать **обычный прямоугольник** (нажать `W` или `A`).

| Изображение на жетоне | Класс |
|---|---|
| Девушка в фуражке (генерал) | `token_general` |
| Мужчина в костюме (финансист) | `token_financier` |
| Девушка с кудрями (культура) | `token_culture` |

Жетоны бывают **круглые** (1 ПО) и **шестиугольные** (3 ПО) — в разметке класс одинаковый.

### 4. Разметка планшета империи (класс `empire_board`)

1. Нарисовать прямоугольник вокруг **всего** планшета.
2. Label: `empire_board`

### 5. Сохранить и экспортировать

1. `File → Save` (сохраняет `.json` в формате LabelMe).
2. `File → Export → YOLO OBB` для получения `.txt` файлов.
   - Или использовать конвертер: `scripts/convert_labels.py` (если добавите).

## Формат файла разметки YOLO OBB

Рядом с каждым `image_001.jpg` должен лежать `image_001.txt`:

```
# class_id  cx  cy  width  height  angle
0 0.423 0.312 0.045 0.612 -12.5
0 0.423 0.670 0.045 0.612 -12.3
1 0.821 0.234 0.078 0.078 0.0
2 0.854 0.234 0.078 0.078 0.0
```

**Поля:**
| Поле | Описание |
|---|---|
| `class_id` | 0=card_strip, 1=token_general, 2=token_financier, 3=token_culture, 4=empire_board |
| `cx`, `cy` | Центр объекта, нормированный к [0, 1] |
| `width`, `height` | Размеры OBB, нормированные к [0, 1] |
| `angle` | Угол поворота в градусах, диапазон [-90, 90) |

## Структура датасета

```
data/dataset/
├── images/
│   ├── train/   # 80% изображений
│   ├── val/     # 15% изображений
│   └── test/    # 5% изображений
└── labels/
    ├── train/   # .txt разметка (YOLO OBB)
    ├── val/
    └── test/
```

Новые реальные фото добавлять в `train/` (или `val/` для проверки).

## Fine-tuning после добавления новых фото

После добавления ≥20 размеченных реальных фото в датасет:

```bash
source recognition/.venv/bin/activate

python scripts/train.py \
  --fine-tune runs/iaww_v1/weights/best.pt \
  --epochs 30 \
  --lr 0.001 \
  --name iaww_v2
```

Флаг `--fine-tune` автоматически:
- Загружает обученные веса (не pretrained)
- Устанавливает пониженный learning rate
- Замораживает первые 10 слоёв для стабильности

## Советы по качеству разметки

- Рисовать рамку **максимально плотно** к объекту — не оставлять лишних отступов.
- Для стопки карт под углом: рамка должна **точно следовать** повороту полоски.
- Не размечать жетоны если они перевёрнуты или значок не виден.
- Оптимальное количество реальных фото для первого fine-tuning: **20–50 изображений**.
