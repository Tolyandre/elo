# Новый концепт - арены и лиги

Арена - это соревновательный режим, который учитывает только избранные партии. 
Каждая арена имеет свои условия, по которым партии попадают в арену. Это могут быть 
ограничения по типам игр, составу участников, датам игр. По арене расчитывается эло рейтинг игроков, который
не зависит от других арен.

Текущий расчёт рейтинга по всем играм и ставкам (текущие названия в коде rating или global_elo) становится
одним из экземпляров арены, которую назовём global. Также создадим арены по каждой игре в отдельности,
которые будут соответствовать расчёту это эло по каждой игре (текущее название в коде game_elo).

В арене есть лиги. Игрок находится в одной из лиг и может двигаться вперёд или назад.
Выполняя определённые условия игроки продвигаются по лигам. В случае неактивности они откатываются.
Пример лиг: бронзовая, серебряная, золотая.
Лиги нужны для того, чтобы игроки сыграли необходимое количество игр, чтобы эло рейтинг начал показывать адекватное значение.

Ещё примеры лиг по устройству:
1. Кубиковая: игрок должен сыграть 10 партий (за каждую партию получает специальное очко "кубик")
2. Мипловая: игрок должен сыграть с разными людьми и получить 10 "мипплов". Каждый соперник даёт 1/(n-1) "мипплов" (n - количество игроков). Причём считается, что уникальный соперник таким образом может дать не более 2 мипплов


## Сущности:

event (событие, созданное пользователем, не может быть изменено при пересчёте истории):
- id
- date
- source ("user", "trigger"): источник события, ввод пользователя или вычисление (наступление условия или срабатывание таймера)
- discriminator ("bet", "match"): тип события, отсылка к соответствующей таблице
Порядок событий определяется полем date. При одинаковых date считается, что событие с меньшим id наступило раньше.

arena:
- id
- settings_id

arena_settings (аналог elo_settings):
- id
- effective_date: дата начала действия настройки. В паре с id образуют первичный ключ. Если для указанного id несколько настроек, то в данный момент действует настройка с наибольшим effective_date среди значений в прошлом
- starting_elo: начальное значение elo для первой партии
- starting_rating: начальный отображаемый рейтинг
- elo_const_k
- elo_const_d
- win_reward



settings (глобальные настройки приложения, независимо от elo_settings. Всегда одна строка):
- global_arena_id: id глобальной арены


games (существующая таблица):
- id
- name
- arena_id: id арены по этой игре. Not null, при создании игры сразу создаётся арена

arena_ratings (аналог таблицы player_ratings. Изменения рейтинга игроков во времени):
- id
- event_id (в результате какого события произошло изменение): TODO: это поле под вопросом. Вместо него date?
- elo (настоящий рейтинг эло игрока на арене)
- rating (отображаемый рейтинг игрока на арене, на период когда сыграно мало партий)
