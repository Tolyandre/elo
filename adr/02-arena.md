# Новый концепт - арены и лиги

Арена - это соревновательный режим, который учитывает только избранные партии. 
Каждая арена имеет свои условия, по которым партии попадают в арену. Это могут быть 
ограничения по типам игр, составу участников, датам игр. По арене расчитывается эло рейтинг игроков, который
не зависит от других арен.

Текущий расчёт рейтинга по всем играм и ставкам (текущие названия в коде rating или global_elo) становится
одним из экземпляров арены, которую назовём global. Также создадим арены по каждой игре в отдельности,
которые будут соответствовать расчёту это эло по каждой игре (текущее название в коде game_elo).

В арене есть лиги. Игрок находится в одной из лиг и может двигаться вперёд или назад.
Выполняя определённые условия игроки продвигаются по лигам. В случае неактивности они откатываются.
Пример лиг: бронзовая, серебряная, золотая.
Лиги нужны для того, чтобы игроки сыграли необходимое количество игр, чтобы эло рейтинг начал показывать адекватное значение.

Ещё примеры лиг по устройству:
1. Кубиковая: игрок должен сыграть 10 партий (за каждую партию получает специальное очко "кубик")
2. Мипловая: игрок должен сыграть с разными людьми и получить 10 "мипплов". Каждый соперник даёт 1/(n-1) "мипплов" (n - количество игроков). Причём считается, что уникальный соперник таким образом может дать не более 2 мипплов


## Сущности и первый шаг к аренам

Сначала трансформируем сущности под новую модель. Начинаем с двух типов арен:
1. global_arena - в единственном экземпляре. Аналог существующего глобального rating
2. game_arena - арена по каждой игре. Аналог game_elo

Таблицы player_ratings и bet_settlement_details превращается в global_arena_settlement, захватывая rating_pay и rating_earn из matches.

Таблица global_arena_settlement: начисление рейтинга в глобальной арене за партию или за ставки (и в будущем вследствие других событий). Подход single table per hierachy.
- id
- player_id
- date
- new_rating
- discriminator ("match", "market"): тип начисления. В будущем появится тип (и соответствующие поля) для ручных корректировок рейтинга
- match_id (только для match)
- market_id (только для market)
- staked: отрицательное число — то, что потрачено/поставлено под риск. Для match = rating_pay (отрицательный), для market = -(сумма ставок)
- earned: положительное число — то, что заработано. Для match = rating_earn, для market = сумма выплат

Таблицы player_ratings, bet_settlement_details - удаляем

Таблица match_scores: результаты партий. Удаляем поля для расчёта рейтинга
- match_id
- player_id
- score

Таблица game_arena_settlement: начисление эло по каждой игре (выносим расчёт из текущей таблицы match_scores)
- id
- game_id
- player_id
- date: дублируем дату из matches для удобства запросов (без лишних JOIN)
- new_rating
- discriminator ("match"): тип начисления, пока всегда match, т.е. в результате партии
- match_id (только для match)
- staked: game_elo_pay (отрицательное число)
- earned: game_elo_earn (положительное число)
