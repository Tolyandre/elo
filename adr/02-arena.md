# Новый концепт - арены и лиги

Арена - это соревновательный режим, который учитывает только избранные партии. 
Каждая арена имеет свои условия, по которым партии попадают в арену. Это могут быть 
ограничения по типам игр, составу участников, датам игр. По арене расчитывается эло рейтинг игроков, который
не зависит от других арен.

Текущий расчёт рейтинга по всем играм и ставкам (текущие названия в коде rating или global_elo) становится
одним из экземпляров арены, которую назовём global. Также создадим арены по каждой игре в отдельности,
которые будут соответствовать расчёту это эло по каждой игре (текущее название в коде game_elo).

В арене есть лиги. Игрок находится в одной из лиг и может двигаться вперёд или назад.
Выполняя определённые условия игроки продвигаются по лигам. В случае неактивности они откатываются.
Пример лиг: бронзовая, серебряная, золотая.
Лиги нужны для того, чтобы игроки сыграли необходимое количество игр, чтобы эло рейтинг начал показывать адекватное значение.

Ещё примеры лиг по устройству:
1. Кубиковая: игрок должен сыграть 10 партий (за каждую партию получает специальное очко "кубик")
2. Мипловая: игрок должен сыграть с разными людьми и получить 10 "мипплов". Каждый соперник даёт 1/(n-1) "мипплов" (n - количество игроков). Причём считается, что уникальный соперник таким образом может дать не более 2 мипплов


## Сущности и первый шаг к аренам

Сначала трансформируем сущности под новую модель. Начинаем с двух типов арен:
1. global_arena - в единственном экземпляре. Аналог существующего глобального rating
2. game_arena - арена по каждой игре. Аналог game_elo

Таблицы player_ratings и bet_settlement_details превращается в global_arena_settlement, захватывая rating_pay и rating_earn из matches.

Таблица global_arena_settlement: начисление рейтинга в глобальной арене за партию или за ставки (и в будущем вследствие других событий). Подход single table per hierachy.
- id
- player_id
- date
- new_rating
- discriminator ("match", "market"): тип начисления. В будущем появится тип (и соответствующие поля) для ручных корректировок рейтинга
- match_id (только для match)
- market_id (только для market)
- staked: отрицательное число — то, что потрачено/поставлено под риск. Для match = rating_pay (отрицательный), для market = -(сумма ставок)
- earned: положительное число — то, что заработано. Для match = rating_earn, для market = сумма выплат

Таблицы player_ratings, bet_settlement_details - удаляем

Таблица match_scores: результаты партий. Удаляем поля для расчёта рейтинга
- match_id
- player_id
- score

Таблица game_arena_settlement: начисление эло по каждой игре (выносим расчёт из текущей таблицы match_scores)
- id
- game_id
- player_id
- date: дублируем дату из matches для удобства запросов (без лишних JOIN)
- new_rating
- discriminator ("match"): тип начисления, пока всегда match, т.е. в результате партии
- match_id (только для match)
- staked: game_elo_pay (отрицательное число)
- earned: game_elo_earn (положительное число)

## Второй шаг

Для новых игроков рейтинг часто падает с каждой партией, пока игрок не займёт актуальное положение в списке игроков (лидеров).
Чтобы игроки не расстраивались из-за падения, предлагается новичкам назначать низкий рейтинг, который будет расти 
пока не сравняется с актуальным значением эло. После чего расчёт рейтинга становится честным и отражает силу игрока (а не количество сыгранных партий). Это мотивирует новичков играть, потому что на старте рейтинг легко даётся, и практически не уменьшается при проигрыше.

В таблицах арен global_arena_settlement и game_arena_settlement разделяем rating (отображаемое в интерфейсе значение) и elo (настоящее расчитанное значение эло):
- new_rating
- rating_staked
- rating_earned
- new_elo
- elo_staked
- elo_earned

При миграции сущестующих делаем эти две величины совпадающими. Вычисление рейтинга после матча или при закрытии рынка ставок теперь состоит из двух расчётов:
1. Elo считается как сейчас, обновляется как сейчас
2. Для расчёта rating формула elo выполняется ещё раз для каждого игрока, только в качестве elo игрока берётся его рейтинг, а elo остальных игроков берётся настоящее.

Такой подход обеспечивает изменение рейтинга (рост) новичков, при этом рейтинг остальных игроков считается независимо 
на основе актуального значения эло новичка. В этом подходе значение rating стремится к значению elo с каждой партией.

В интерфесе добавления партии есть предрасчёт эло на клиенте - его убираем, т.к. точный расчёт возможен только на сервере.
Значение эло в результатах партии `/match`, на `/matches`/, `/players`, значение на графике `/player` теперь должно замениться на значение rating. Однако, значение для статистики по успешным играм и "я понял как играть" (неуспешные игры) должно использовать реальное значение elo. Расчёт в интерактивном калькуляторе в справке `/help` не учитывает rating, т.е. работает как сейчас только по значению elo.


Чтобы управлять количеством игр, необходимых для достижения rating актуального elo, значения коэффициентов D и K также могут отличаться при расчёте rating. Также могут быть другие изменения в правилах расчёта. Значения коэффициентов и эти правила будут определяться лигой, в которой игрок находится внутри арены.

Для глобальной арены такие лиги:

1. Лига новичков (Newbie). Рейтинг не может уменьшаться, если он должен уменьшится, то значения rating_staked и rating_earned 
переопределяются rating_staked = 0, rating_earned = 1 (т.е. даже при поражении rating увеличивается на 1).
Для перехода в следующую лигу нужно набрать 500 (условно, стартовое значение rating равно 0)

2. Лига любителей. Состоит из игроков которые продолжают играть в настольные игры или играют нерегулярно.
Расчёт rating производится по обычным правилам (может уменьшаться). Возврат в лигу новичков не производится.
Чтобы перейти в высшую лигу, нужно сыграть 20 партий за последние полгода, причём хотя бы 3 должны быть за последние 2 месяца

3. Высшая лига. Состоит из игроков, которые часто играют. Игроки возвращаются в лигу любителей, как только условие пребывания
в высшей лиге перестаёт выполняться. Расчёт rating производится по обычным правилам.

Для арены game_arena (по играм) лиги:

1. Лига новичков
2. Лига любителей

Высшей лиги нет, т.к. чаще всего нет игроков, которые постоянно играют только в эту игру.

На странице `/players` выводится список игроков с рейтиногом глобальной арены. Сейчас список разделён на две части, теперь вместо этого будет выводиться три лиги. Соответственно, список `/game` должен быть разделён на две части, по количеству лиг в game_arena.

Для хранения настроек новых лиг используется таблица elo_settings. Новые столбцы:
- starting_rating: начальное значение rating для новых игроков. Значение: 0
- newbie_league_goal: значение рейтинга для перехода из лиги новичков в следующую лигу. Значение: 500
- newbie_league_elo_const_k: значение коэффициента k для расчёта рейтинга при нахождении в лиге новичков. Значение: 64
- newbie_league_elo_const_d: значение коэффициента d для расчёта рейтинга при нахождении в лиге новичков. Значение: 400
- (придумать название): количество партий, которые нужно сыграть за последние полгода, для перехода в высшую лигу. Значение: 20
- (придумать название): количество партий, которые нужно сыграть за последние 2 месяца, для перехода в высшую лигу. Значение:3

На интерфейс в админке пока не выводим новые поля elo_settings. При изменении через админку сохраняем предыдущие значения полей, которые не проброшены в UI. Для значения win_reward не вводим аналог для newbie_league.

Обработка перехода в следующую лигу должна происходить при расчёте или пересчёте рейтинга каждой партии или результатов закрытия рынка ставок. Для этого вводим в таблицы арен новое поле (тип лиги)
