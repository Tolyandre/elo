# Кемпы (турниры, выезды на совместные игры)

Игроки собираются вместе и играют в настольные игры несколько дней подряд - 
проводят игровые кемпы. Назовём это tournament.

Игроки играют между собой, поэтому при занесении партий нужна возможность искать игроков среди 
группы игроков, участвующих в кемпе (турнире). Поиск среди всех игроков сложен,
а поиск по клубам не помогает, потому что на выездных играх присутствуют игроки разных клубов.

Также нужна статистика по играм, сыгранным на кемпе.

## Доработки

Вводим новую сущность tournament для обозначения кемпов и турниров.
- id
- name
- дата начала
- дата окончания

Tournament соотносится с игроками players многие-ко-многим.

Если при создании партии текущая дата попадает в интервал действия, то в форме появляется галочка
с названием турнира. Партия может относится к нескольким турнирам, но это будет редко.
Галочка по умолчанию проставлена, в случае если игрок текущего пользователя входит в турнир.

В выпадающем списке игроков в случае проставления галочки появляется новая секция с названием турнира,
в которую входят все участвующие игроки. Меняется порядок следования секций:
1. Недавние
2. Турниры в алфавитном порядке, для которых поставлена галочка
3. Клубы в алфавитном порядке, но в первую очередь те клубы, в которые входит игрок текущего пользователя
4. Без клуба

Помимо этого, в секцию "Недавние" должны входить недавние партии с игроком текущего пользователя,
а не любые последние партий.

При сохранении партии на сервер если партия относится к турниру, то все участвовавшие игроки
добавляются в этот турнир. Если при редактировании партии игрок меняется на другого, то удаления
из участников турнира не происходит.

При редактировании партии галочка с возможностью добавить в турнир появляется и исчезает при 
изменении даты партии.

При просмотре партий в каротчке партии должна быть отметка с названием турнира после списка игроков.

Возможность создавать турниры в режиме офлайн не нужна, но добавление партий, относящихся к турниру, 
нужно. Для этого список турниров и участников должен быть предзагружен (как обычный список участников сейчас).

В меню добавляем новый пункт "Кемпы и турниры", который будет вести на список турниров.
Список турниров отсортирован по дате начала, сначала новые. Даты выводить не нужно.
В списке турниров должна быть возможность создать новый турнир или просмотреть или отредактировать существующий (при наличии права редактирования).
Страница создания и редактирования турнира позволяют указать имя, даты, и выбрать игроков-участников.
При редактировании нельзя удалить участника, если он участвовал хотя бы в одной партии турнира.
Также нельзя изменить даты так, чтобы даты уже сыгранных партий вышли за диапазон.

Страница просмотра турнира отображает название и статистику по игрокам. Должен выводиться список участников, количество партий, количество первых, вторых, третьих и четвёртых мест участника (с изображением медалей, уже использющихся для ранжирования).
Сортировка  сначала по количеству первых мест, потом вторых, третьих, четвёртых, затем
числу игр и алфавиту

Если есть активный турнир, то страница `/palyers` в верху списка должна содержать на него ссылку,
поскольку это главная страница, а турнир - целое событие.
