# Изменение рейтинга новичков

При стремлении рейтинга новичков к реальному эло есть недостатки. Составляющая штрафа за проигрыш
rating_staked увеличивается в середине пути, когда gap (gap = |elo - rating|) равен примерно D или 4*tau.
В начале пути и в конце (gap=0) составляющая штрафа в порядке, равна обычному штрафу по формуле эло. 
Похоже что это получается потому, что штраф в эло определяется логистической функцией, а коэффициент
Krating, используемый для стремления рейтинга к эло, определяется экспоненциальной (экспоненциальное затухание).

Вторая проблема: граница отсечения рейтинга новичков и любителей выбрана произвольным значением, которое ничего означает. В в момент её пересечения рейтинг всё ещё не доганяет эло.

Предлагаются следующие изменения:
1. Для стремления рейтинга к эло отказываемся от изменения коэффициента K (Krating). Поэтому значение rating_max_k, rating_k_taue в elo_settings больше не нужны.
По-прежнему считаем новый рейтинг игрока, используя его существующий рейтинг и эло соперников, с настоящими значениями D и K. Однако, будем масштабировать составляющую elo_earned.

Обычные значения elo_earned находятся в диапазоне [0, K] (от 0 до K). Нам нужна функция, которая будет мапить
elo_earned из этого диапазона в [newbie_league_earned_min, newbie_league_earned_max]. При приближении рейтинга к эло этот диапазон должен стремиться снова к [0, K]. 
Предлагается использовавшаяся для Krating функция экспоненциального насыщения с коэффициентом newbie_league_earned_tau.

Значения:
- newbie_league_earned_min = 2
- newbie_league_earned_max = 64
- newbie_league_earned_tau = 100

2. Больше не нужен "зажим" (clamp) для лиги новичков, который делает всегда +1 при проигрыше, если рейтинг меньше 500 (newbie_league_goal). Эту роль выполняет newbie_league_earned_min, поэтому пока рейтинг мал, игры будут в плюсе даже при проигрыше.

Новое условие для перехода из лиги новичков в лигу любителей (или высшую): когда gap <= newbie_league_goal_gap. Значение newbie_league_goal_gap = 16. Переход не отменяет формулу rating, а только фиксирует возможность игрока занимать места в Лиге Любителей или в Высшей Лиге.

Для вывода в интерфейс информации по условиям перехода в слудующую лигу нужно расчитать примерное количество побед, необходимых для выполнения условия по gap. Для этого нужно вывести аналитическую формулу.

3. Для арены по играм нужно отдельное значение starting_rating, потому что партий в каждую конкретную игру гораздо меньше, чем партий в глобальной арене. Поэтому разделяем значения:

- starting_rating_global_arena = 0
- starting_rating_game_arena = 900

Лига Новичков для арены по играм в остальном расчитывается также, как Лига Новичков на глобальной арене.

## Применение изменений

Мы меняем существующую формулу, поэтому произведём перерасчёт уже записанных партий с помощью псевдо редактирования и сохранения самой первой партии.
