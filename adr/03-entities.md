# Рефакторинг сущностей

Для обобщения сущностей и нормализации данных

## Сущности:

events (событие, созданное пользователем, не может быть изменено при пересчёте истории):
- id
- date
- user_id
- source ("user", "trigger"): источник события, ввод пользователя или вычисление (наступление условия или срабатывание таймера)
- discriminator ("bet", "match"): тип события, отсылка к соответствующей таблице
Порядок событий определяется полем date. При одинаковых date считается, что событие с меньшим id наступило раньше.
Согласно Table-per-type таблицы bets и matches расширяют events
